# Возможности

Ниже перечислены текущие возможности библиотеки

## Поддерживаемые типы объектов метаданных

На данный момент библиотека предоставляет возможность только чтения информации о метаданных конфигурации и расширения.

### Общие наборы данных

В зависимости от типа объекта метаданных читаются общие наборы данных

- Базовые атрибуты
  - Наименование
  - Уникальный идентификатор
  - Синонимы
  - Комментарий
  - Принадлежность к расширению
  - Ссылка (mdoRef)
  - Принадлежность к подсистеме
- Дочерние элементы (для тех, у кого они могут быть)
  - Формы
  - Шаблоны
  - Команды
- Модули (для тех, у кого они есть)

Более подробный список общих можно увидеть в классах пакета `mdo` с префиксом `Abstract`, от которых унаследован интересущий класс типа метаданных.

### Данные реквизитов

Для типов метаданных, имеющих реквизиты, происходит чтение всех "прикладных" реквизитов, т.е. добавленных разработчиком, а именно 
- Реквизиты
- Измерения
- Ресурсы
- Табличные части
- Колонки журналов
- и т.д.

Для каждого реквзита читается базовый набор данных, как и для объектов метаданных, а также признаки индексированности. Кроме того, для измерений происходит чтение дополнительного набора специализированных атрибутов.

### Данные форм

Для всех форм объекта происходит чтение базовых атрибутов, типа формы (обычная или управляемая).
В обычных формах больше информации не получить, но для управляемой формы происходит чтение дерева элементов с типами, реквизитов и обработчиков.

### Данные макетов

При чтении макетов, кроме базовой составляющей, выполняется определение типа макета (текстовый макет, табличный документ, двочиные данные и т.д.). Для макетов со схемой СКД реализовано чтение данных запроса СКД.
